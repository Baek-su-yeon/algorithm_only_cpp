# 📜 THE RULES

### 1. 문제 선정
* **매일** 문제를 선택
* `main` 브랜치에 직접 접속하여 `TodayProblem.md` 파일에 문제에 URL 적기

### 2. 문제 풀기
* 본인의 이름을 딴 **개별 브랜치**를 생성한 후, 해당 문제 폴더 내에 코드를 작성

### 3. 명명 규칙
* **파일 명:** `SWEA_문제번호_이름.cpp` (문제 번호가 없을 시 문제명으로 대체)
    * *예: `SWEA_1234_성현.cpp`*
* **커밋 메시지:** 파일 명과 동일하게 작성

### 4. 보고 및 승인
* PR(Pull Request) 시 **Label** 기능을 사용하여 해당 알고리즘 태그를 반드시 부착 (DFS, DP, Greedy 등)
* **Assignees**는 '본인'으로 설정
* 다음 날 merge 후 `archive` 폴더로 문제 이동

### 5. 고나리법 (Code Review)
* 모든 팀원의 코드에는 반드시 **최소 1개 이상의 고나리질(리뷰)**
* 논리적 허점, 가독성, 혹은 날카로운 지적도 적극 권장


## 👑 SPECIAL ENTITY (특별 존재)

### **Observer 역할**
* **자유 문제 풀기**
* Observer: 나 외에 모든 팀원
